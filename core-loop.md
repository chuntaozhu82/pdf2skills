## 功能说明
本技能用于分析和生成游戏的核心循环，包括：
- 玩家行为序列
- 资源流转路径
- 关键决策点
- 核心「一局游戏」的节奏

## 适用场景
- 从 0 设计新游戏玩法
- 对已有玩法做核心循环重构/诊断
- 为后续数值和平衡设计提供结构输入

## 输入参数
- 游戏类型（如：Roguelike、塔防、卡牌、SLG）
- 核心资源（如：生命、能量、金币、卡牌）
- 核心动作节点（如：探索、战斗、建造、对话）

## 处理流程（伪代码示意）

```python
# 定义核心循环构建步骤
1. 收集游戏类型 & 目标体验
2. 列举玩家可能的主要动作
3. 将动作按时间顺序排列成循环
4. 为每一步标注：
   - 输入资源
   - 输出资源
   - 关键决策点
5. 将结果转化为 mermaid 流程图描述
6. 抽取资源列表，形成资源表（名称/获取/消耗/初始值）

## 输出格式
1. 一段不超过 100 字的核心循环文字概述  
2. 一个 mermaid 流程图代码块，例如：
graph TD
    A[抽卡] --> B[出牌]
    B --> C[结算]
    C --> D[资源管理]
    D --> A
3. 一张资源系统表（Markdown 表格）：
资源	获取方式	消耗方式	初始值
能量	每回合 +1	出牌	3
生命	防御成功	受伤	10
